## 1. 实验范围

两个环境使用相同网格：

```text
alpha       = [0.5, 1, 2, 4, 8, 16, 32]
lambda_ctrl = [0, 0.001, 0.005, 0.01, 0.05, 0.1, 0.5]
seed        = 2025
```

每个环境 49 次训练和 49 次 DWM-only rollout，共 98 次训练与 98 次 rollout。所有成功训练的点
都进入闭环 L2 表，不做 seed 筛选，也不根据单帧指标提前裁掉网格点。

## 3. DWM-only 与删除旧中间产物的收益

`transition_dataset.npz` 的 `(s,a,s')` 数据服务于 learned dynamics，但当前 pipeline 使用
`dynamic.py` 的解析 dynamics，仓库内没有该文件的消费者。每次旧 sampling 为它执行
`2400 × 30 = 72,000` 次真实渲染；完整 98 点消融会重复这一成本。DWM-only sampling 跳过真实
renderer 和 transition 写盘，只保留本轮需要的 decoder 闭环轨迹。

旧 `metadata.json` 会被不同脚本覆盖，而且配置已经由 git 管理。现在关键 rollout 溯源直接保存到
NPZ 的 `variant` 和 `decoder_weights` 字段，并由断点验证器核对，来源信息更靠近实际产物。

## 4. L2 定义

CartPole 对四维完整 state 使用普通 L2。Pendulum 对 theta 使用最短圆周差，再与 omega 一起计算
full-state L2。validation 和 test 均报告：

- `mean_step_l2`；
- `final_l2`；
- `max_l2_mean`；
- `max_l2_p95`。

validation 用于选择超参数；test 只用于最终候选报告，不反向参与调参。L2 仍是描述性模型比较指标，
不是论文 Theorem 1 的 L1 non-conformity score。
